## TCP 协议

### 七层协议

应表会传网数物

### 三次握手，四次挥手，TIME_WAIT

为什么有TIME_WAIT，是因为客户端收到服务器的FIN消息时，回复一个ACK，直接进入Closed状态，没有进入time_out状态。如果ACK丢失，则服务器会重发FIN，就会影响客户端接下来的连接，所以默认等待2个MSL时间。

### 滑动窗口

选择重传。接收方将收到正确的先存放到缓冲区，然后要求发送端只发送错误的帧，接收正确后，从缓冲区取出正确的帧，按照正确的排序递交给上层。

### 拥塞控制

接收方就是在ACK中加入接收缓冲区的大小。

### 顺序控制

发送数据加校验和跟序号。

### 黏包

需要在应用层加入长度控制。

### 超时重传

发送一个包就启动一个定时器，如果超时收到ACK就会重传

### 快速重传

接收方接收到后面序号的数据先到达，则立即向发送方发送重复三次ACK。发送方接收到连续3个相同序号的ACK，则理解重传不等超时

### Socket 选项

| 选项 | 说明 |
|------|------|
| SO_KEEPALIVE | 打开keepalive |
| TCP_KEEPDILE | 设置连接上如果没有数据发送的话，多久后发送keepalive探测分组，单位是秒 |
| TCP_KEEPINTVL | 前后两次探测之间的时间间隔，单位是秒 |
| TCP_KEEPCNT | 关闭一个非活跃连接之前的最大重试次数 |
| TCP_NODELAY | 小包也发送，不在缓冲区缓存，控制Nagle算法 |
| SO_LINGER | 优雅关闭socket 保证缓冲区中的数据发送出去 |
| SO_SNDBUF | 发送缓冲区 |
| SO_RCVBUF | 接收缓冲区 |

## 网络基础

### DNS

`/etc/resov.conf`

### 网卡信息

`/etc/sysconfig/network-scripts/xxx` 下

- 修改网关: `/etc/sysconfig/network`
- 修改IP: `ifconfig eth0 192.168.0.2 netmask 255.255.255.0`
- 修改网关: `route add default gw 192.168.0.1 dev eth0`
- 修改DNS: `/etc/resov.conf`
- 修改host name: `/etc/sysconfig/network`
- 修改网卡: `/etc/udev/rules.d/70-persistent-net.rules`

## tcpdump

```bash
tcpdump -i any -s 0 -v -w test.pcap
```

### 常用参数

| 参数 | 说明 |
|------|------|
| `-i eth1` | 只抓经过接口eth1的包 |
| `-t` | 不显示时间戳 |
| `-s 0` | 抓取完整的数据包 |
| `-c 100` | 只抓取100个数据包 |
| `dst port ! 22` | 不抓取目标端口是22的数据包 |
| `src net 192.168.1.0/24` | 数据包的源网络地址为192.168.1.0/24 |
| `-w ./target.cap` | 保存成cap文件 |
| `-v` | 详细显示指令执行过程 |

### 示例

```bash
tcpdump -i enp4s0f0 -s 0 -v -w test.pcap
tcpdump -i fm1-mac9 -s 0 -v -w test.pcap
```
