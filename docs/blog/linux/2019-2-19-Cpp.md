## static 和 const

### Static

1. **隐藏功能**
2. **全局静态区**：刚开始运行就初始化唯一一次，默认初始化为 0
3. **静态局部变量**：生命周期为整个程序的生命周期
4. **类成员**：
   - 属于类而非对象，没有this指针
   - 不能将静态成员函数定义为虚函数
   - 静态成员函数主要用来访问静态数据成员而不能访问非静态成员

### const

常量。编译器保护那些不希望修改的参数。

**优点：**
- 编译器进行检查
- 节省空间，存放在符号表中，第一次使用时为其分配内存，在程序结束释放

**const修饰指针变量：**
- `const在*左侧`：指针所指数据是常量，不能通过解引用修改；指针本身是变量
- `const在*右侧`：指针本身是常量，不能指向其他地址；所指数据可修改
- `const在*两侧`：指针和指针所指数据都不能修改

## 强制类型转换

### static_cast
- 上转安全，下转不安全
- 基本类型转换
- 空指针转换成非空指针
- 把任何类型转换成 void 类型
- 不能转换 const、volatile

### dynamic_cast
- 用于类层次间的上行转换和下行转换
- 可以进行类之间的交叉转换
- 下行转换时具有类型检查功能，比 static_cast 更安全
- 转换失败时：指针返回0，引用抛出 std::bad_cast 异常

### const_cast
- 修改类型的 const 或 volatile 属性
- 常量指针转成非常量指针
- 常量引用转成非常量引用

### reinterpret_cast
- 低级转型，指针和整数互转
- 结果取决于编译器，不可移植

## 类的大小

```cpp
// 1. 空类
class A {};
sizeof(A);  // 1

// 2. 虚函数类
class A { virtual void Fun(); };
sizeof(A);  // 4 (虚函数指针)

// 3. 普通数据成员
class A { int a; char b; };
sizeof(A);  // 8 (内存对齐)

// 4. 静态数据成员
class A { int a; static int b; };
sizeof(A);  // 4 (不计入静态成员)

// 5. 普通成员函数
class A { void Fun(); };
sizeof(A);  // 1
```

## 线程同步

### 四种线程同步方式

1. **互斥锁 (Mutex)**

```cpp
#include <mutex>

std::mutex mtx;

void critical_section() {
    std::lock_guard<std::mutex> lock(mtx);
    // 临界区代码
}
```

2. **条件变量 (Condition Variable)**

```cpp
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
std::queue<int> queue;

void producer() {
    std::lock_guard<std::mutex> lock(mtx);
    queue.push(42);
    cv.notify_one();
}

void consumer() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, [] { return !queue.empty(); });
}
```

3. **读写锁 (Read-Write Lock)**

```cpp
#include <shared_mutex>

std::shared_mutex rw_mutex;

// 读操作（多个线程可以同时读）
void read_data() {
    std::shared_lock<std::shared_mutex> lock(rw_mutex);
}

// 写操作（独占访问）
void write_data() {
    std::unique_lock<std::shared_mutex> lock(rw_mutex);
}
```

4. **信号量 (Semaphore)**

```cpp
#include <semaphore>

std::counting_semaphore<3> sem(3);

void worker() {
    sem.acquire();
    // 临界区代码
    sem.release();
}
```

### 自旋锁

```cpp
#include <atomic>

class SpinLock {
    std::atomic_flag flag = ATOMIC_FLAG_INIT;
public:
    void lock() {
        while (flag.test_and_set(std::memory_order_acquire));
    }
    void unlock() {
        flag.clear(std::memory_order_release);
    }
};
```

## STL

### 组件
- **容器**：vector, list, set, map
- **算法**：sort, copy, erase
- **迭代器**：注意 vector 的迭代器失效问题
- **仿函数**
- **容器适配器**
- **空间适配器**

### 容器特点

| 容器 | 底层数据结构 | 特点 |
|------|--------------|------|
| vector | 数组 | 快速访问 |
| list | 链表 | 快速增删 |
| set | 红黑树 | 有序，不重复 |
| map | 红黑树 | 有序，不重复 |
| hash_set | hash表 | 无序，不重复 |
| hash_map | hash表 | 无序，不重复 |

## GCC 和 CMake

### GCC 常用命令

```bash
# 编译
gcc -c *.c

# 生成静态库
ar -rcs libn9m.a *.o
```

### CMake 常用命令

```cmake
cmake_minimum_required(VERSION 2.8)
project(demo)

add_executable(main main.c)

aux_source_directory(dir var)
set(SRC_LIST ./main.c ./testFunc1.c ./testFunc.c)

include_directories(dir)

add_subdirectory(src)

add_library(testFunc_shared SHARED ${SRC_LIST})
add_library(testFunc_static STATIC ${SRC_LIST})

set(LIBRARY_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/lib)

find_library(TESTFUNC_LIB testFunc HINTS ${PROJECT_SOURCE_DIR}/testFunc/lib)

target_link_libraries(main ${TESTFUNC_LIB})

add_compile_options(-std=c++11 -Wall)

option(MYDEBUG "enable debug" OFF)
if(MYDEBUG)
    add_executable(main2 main2.c)
endif()
cmake .. -DMYDEBUG=ON
```
